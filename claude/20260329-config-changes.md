# Configuration Changes Summary

## TypeScript: Project References Setup

Replaced a single `tsconfig.json` (and an unused `tsconfig.server.json`) with a three-file project references structure. This gives the IDE environment-specific type-checking -- server files no longer have access to `document`/`window`, and client files don't see Node APIs.

### tsconfig.json (root)

Compiles nothing. References the other two configs.

**Before:**
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "jsx": "react-jsx",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "erasableSyntaxOnly": true,
    "noFallthroughCasesInSwitch": true,
    "noUncheckedSideEffectImports": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

**After:**
```json
{
  "references": [
    { "path": "./tsconfig.app.json" },
    { "path": "./tsconfig.server.json" }
  ],
  "files": []
}
```

### tsconfig.app.json (new)

Client code. Extends the community `@tsconfig/vite-react` base which provides: `target: ES2022`, `module: ESNext`, `moduleResolution: bundler`, DOM libs, JSX, `strict`, `verbatimModuleSyntax`, `noEmit`, and all linting options.

```json
{
  "extends": "@tsconfig/vite-react/tsconfig.json",
  "compilerOptions": {
    "composite": true,
    "declarationMap": true
  },
  "include": ["src/client/**/*"]
}
```

- `composite` -- required for project references
- `declarationMap` -- enables "Go to Definition" to navigate to `.ts` source across project references (needed when shared types are added)

### tsconfig.server.json (updated)

Server code. Extends the community `@tsconfig/node24` base which provides: `target: es2024`, `module: nodenext`, `moduleResolution: node16`, `strict`, `esModuleInterop`, `skipLibCheck`.

**Before:**
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "lib": ["ES2022"],
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "outDir": "dist/server",
    "rootDir": "src/server"
  },
  "include": ["src/server/**/*"],
  "exclude": ["node_modules"]
}
```

**After:**
```json
{
  "extends": "@tsconfig/node24/tsconfig.json",
  "compilerOptions": {
    "composite": true,
    "declarationMap": true,
    "outDir": "dist/server",
    "rootDir": "src/server",
    "verbatimModuleSyntax": true,
    "moduleDetection": "force",
    "erasableSyntaxOnly": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noFallthroughCasesInSwitch": true,
    "noUncheckedSideEffectImports": true
  },
  "include": ["src/server/**/*"]
}
```

### Settings removed as redundant

These were in the old config but are no longer needed:

| Setting | Why removed |
|---------|-------------|
| `esModuleInterop` | Provided by both base configs |
| `forceConsistentCasingInFileNames` | Default in TypeScript 5.x+ |
| `resolveJsonModule` | Automatic with `moduleResolution: "bundler"` |
| `isolatedModules` | Superseded by `verbatimModuleSyntax` |
| `exclude: ["node_modules"]` | Excluded by default |

## Biome: Linter and Formatter

Added Biome as a single tool replacing both ESLint and Prettier (neither were previously configured).

### biome.json (new)

```json
{
  "$schema": "https://biomejs.dev/schemas/2.4.9/schema.json",
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true
  },
  "files": {
    "includes": ["**", "!!**/dist"]
  },
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "indentWidth": 2
  },
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true
    }
  },
  "javascript": {
    "formatter": {
      "quoteStyle": "single"
    }
  },
  "assist": {
    "enabled": true,
    "actions": {
      "source": {
        "organizeImports": "on"
      }
    }
  }
}
```

- Formatter configured to match existing code style (2-space indent, single quotes)
- Recommended lint rules enabled
- Automatic import sorting enabled
- Respects `.gitignore` and excludes `dist/`

### package.json scripts added

```json
"lint": "biome check src/",
"lint:fix": "biome check --write src/",
"format": "biome format --write src/"
```
